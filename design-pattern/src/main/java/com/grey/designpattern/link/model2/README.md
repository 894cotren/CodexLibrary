# 责任链-支持多链（不需要开多例模式）

我感悟下来的理解就是，把model1的下一个结点的链接关系解耦出来，通过一个链表来维护了。通过Node来维护前后关系，结点只专注自身处理逻辑

然后再把抽出来解耦出来的结点内容，作为一个乘客，乘坐在链表上，不需要关心其他，前后关系有链表的Node来解决。

这也是为啥Node里面还封装了一层一样(不，应该叫做Node里面提供了一个被封装的乘客位置，泛型item)，就像AQS里面的线程排队进入队列再封装一下结点一样。


所以这里的链表的实现都不太重要，重要的是链表的结点Node预留了核心逻辑存放的位置；
而链表本身，只是维护了结点之间的关系，以及链表本身的一些属性，比如链表的名称，链表的长度等等。

然后改链表作为一个列车模板，被各种责任链创建出来，里面装不同的责任结点乘客。
